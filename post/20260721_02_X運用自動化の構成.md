# 投稿文

X投稿の運用を2週間かけて全自動化した。

投稿・指標収集・週次分析まで、ほぼ無料枠で回っている。

人間がやるのは「日曜にレビューして承認」と「返信」だけ。

個人開発の題材としても学びが多かったので、構成をスレッドで残します👇

===

①投稿の仕組み

cron-job.org → GitHub Actions → X API の3段構成。

投稿文はリポジトリにMarkdownで管理し、frontmatterのstatusが「承認済み」のものだけを定時に自動投稿。投稿後はtweet_idを自動で書き戻す。

===

②なぜ外部スケジューラを挟むか

GitHub Actionsのcron起動は混雑時間帯だと数時間遅れる（実測で最大+5時間超）。

cron-job.orgからworkflow_dispatchを叩くと起動は数秒。定時性が要る自動化はこの構成が安定だった。

===

③指標収集の工夫

手動投稿分はtweet_idが分からない問題を、タイムラインAPIと本文の突合で解決。

改行や句読点は投稿時に手直しされがちなので、文字と数字だけ残して比較するのがコツだった。

===

④週次分析

毎週日曜にGitHub Actions内でClaude CodeのCLIを起動し、指標CSVから分析レポートを自動生成。

サブスクのOAuthトークンで動かせるのでAPI課金なし。日曜昼には「読むだけ」の状態になっている。

===

まとめ：自動化して分かったのは、伸ばす作業（返信・会話）こそ人間の仕事だということ。

機械にできることを機械に渡すと、続けることが一気に楽になる。

同じ構成を組みたい方がいたら気軽にどうぞ。
