  4. 補助ルート: zenn.devトレンド 記事著者のXリンク／Xで #個人開発 を検索→「最新」タブ
  5. 見つけた人を:
  6. analysis/reply_targets.html のB表に追記（@ID・一言メモ・規模・返信で足せる自分の価値）
  7. Xの非公開リストに追加：自分のプロフィール → リスト → 作成 →「返信先」→ 非公開ON → メンバー追加

① 平日ルーティン：返信活動（15〜30分）

通勤中・昼休みでOK。時間割：

迷ったら：アクション判断フロー（デフォルトは返信）

良い投稿を見つけた
├─ 自分の経験・データ・問いを足せる → 返信する（第一選択）
├─ 足せる一言はあるが会話にはならなそう → 引用RT（週1〜2の枠内で）
├─ 何も足せないが良かった → いいねだけ
└─ フォロワーに丸ごと見せたい → 無言リポスト（例外的に）

身の丈フィルター（2026-07-12の運用で学んだこと）：大物の完成された投稿には返信の余地がない。コメントが出ないのは自分のせいではなく相手選びの問題。探すのは「半年前の自分と同じ場所にいる人」「学習ログ・途中経過を出している実務者」＝読んで7割わかり3割学びがある投稿。そこが一番言葉が出る。返信に「意見」は不要——心の声（あるある／へぇ／ほんとか？／わからん）を型に翻訳するだけでよい。⚠️ ただし「初心者向け」に寄りすぎて #駆け出しエンジニア 系の空間に入らないこと（信頼と逆行）。結果（フォロワー・いいね）は急がない：KPIは月+50〜150で設計済み。

行動 成長への効果 適量
返信（価値を足すリプライ） ◎ 本体。著者のフォロワー全員に露出＋往復が最重み 平日5〜8件
引用リポスト（一言つき） ○ 補助 週1〜2件まで（多いと自分の発信が埋もれる）
無言リポスト △ 自分の発言ゼロ＝人となりが伝わらない 基本しない
いいね △ 関係の潤滑油 気軽に。ただし返信の代わりにはならない

(1) 通知チェック — 3分

  • 自分の投稿への返信には全部返す（24時間以内、投稿後30分は最優先）
  • いいねだけくれた人は、相手のプロフィールを一瞬見て、合いそうなら投稿にいいねを返す

(2) 本命への返信 5〜8件 — 15〜20分（ここが本体）

  • Xの非公開リスト「返信先」を開く → 当日投稿している人を上から探す
  • 投稿直後（1時間以内）ほど良い。上位返信の位置を取れる
  • そのまま使える返信の型（自分の実体験を1つ入れるのがコツ）:
型 例
経験共有 「同じ場面ありました。自分は◯◯で解決したんですが、△△のケースはどうされてますか？」
深掘り質問 「◯◯の部分が気になりました。△△ではなくこちらを選んだ決め手って何かあったんでしょうか」
データ・補足 「これ◯◯でも同様でした。実測だと△△だったので、規模によって変わりそうですね」
節度ある別視点 「なるほどです。逆に◯◯の場合は△△が効いた経験があって、使い分けかもしれません」
  • NG: 「すごい！」「参考になります」だけ／絵文字のみ／自分の宣伝。著者が「返信し返したくなるか」が基準
  • 相手から返信が来たら会話を1〜2往復続ける（この往復がアルゴリズム上いちばん重い）

(3) 引用RT＋ネタ集め — 3〜5分

  • 良い技術ポストを1〜2件、自分の一言（経験・意見）を添えて引用RT
  • タイムラインで気づいたこと・今日ハマったことをメモ → ②のライブネタや次バッチの種にする

② ライブネタ投稿：週1（5分）

  • その週に実際あったことを1本手動投稿：今日ハマったバグ、いま作っているもの、学習の途中経過
  • 完成した話より途中の話。「〜で迷っている」「〜という理解で合ってるだろうか」のような 問いかけで終えると返信が来やすい
  • 予約枠と同じ日でも構わない（1日2本になっても問題ない）
  • 投稿後の記録: post/ に md 化して status: 予約済み で push しておく → 日曜の自動処理（reconcile）が tweet_id を拾い、指標収集に乗る。整形は /create-x-post に頼めばよい

③ 日曜バッチ（20〜30分）

  1. git pull → analysis/ の最新週次レポートを読む（日曜11時に自動生成済み）
  2. レポートの改善提案を見て、翌週の出し方を決める
  3. 在庫レビュー：次に出す下書きを確認（現在庫は7/25で尽きる → /create-x-post で補充）
  4. 出すものの frontmatter を status: 承認済み＋scheduled_at（9/12/15/18/20/22時をばらす）にして push ＝予約完了
  5. reply_targets.html の手入れ：反応をくれた人を追加、2週間反応がない人を入れ替え

④ 月1：Premiumアナリティクス確認（10分）

  • Xアプリ → プレミアム → アナリティクス で プロフィール訪問数（アクセス）とフォロワー増減を確認 （この2つはAPIで取れないため自動レポートに含まれない）
  • 「プロフィール訪問は増えているのにフォローが増えない」→ bio・固定ツイートを見直す合図（account.html の確定版と見比べる）
