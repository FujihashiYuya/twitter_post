# X運用 実践プレイブック（投稿以外にやること）

投稿は全自動化済み（cron-job.org → GitHub Actions）。**アカウントの成長は投稿ではなく、ここに書く手動の活動から生まれる**（小規模アカウントの成長の約半分は「返信」から）。
作成: 2026-07-05 ／ 戦略の根拠は `x_strategy.md`、このファイルは「手順書」。

## ⓪ 初回タスク：本命リストを10人埋める（30分）

`reply_targets.md` の「B. 返信ターゲット」は今空欄。ここを埋めないと①が始められない。

1. Xアプリで **@minorun365 / @uhyo_ / @jnchito** など（`reply_targets.md` のA表）を開く
2. 各人の最新ポストの**リプ欄**を見る → 「一言じゃない、ちゃんとした返信」をしている人のプロフィールを開く
3. **フォロワー500〜3,000 ＋ Web/AI/個人開発の発信 ＋ リプ欄が会話になっている** ならリスト入り
4. 補助ルート: [zenn.devトレンド](https://zenn.dev) 記事著者のXリンク／Xで `#個人開発` を検索→「最新」タブ
5. 見つけた人を:
   - `analysis/reply_targets.md` のB表に追記（@ID・一言メモ・規模・返信で足せる自分の価値）
   - **Xの非公開リストに追加**：自分のプロフィール → リスト → 作成 →「返信先」→ **非公開ON** → メンバー追加
6. フォローもするなら1日5〜10人まで（乱発はシャドウバン要因）

## ① 平日ルーティン：返信活動（15〜30分）

通勤中・昼休みでOK。時間割：

### (1) 通知チェック — 3分
- 自分の投稿への返信には**全部**返す（24時間以内、投稿後30分は最優先）
- いいねだけくれた人は、相手のプロフィールを一瞬見て、合いそうなら投稿にいいねを返す

### (2) 本命への返信 5〜8件 — 15〜20分（ここが本体）
- Xの非公開リスト「返信先」を開く → **当日投稿している人**を上から探す
- 投稿直後（1時間以内）ほど良い。上位返信の位置を取れる
- **そのまま使える返信の型**（自分の実体験を1つ入れるのがコツ）:

| 型 | 例 |
|----|-----|
| 経験共有 | 「同じ場面ありました。自分は◯◯で解決したんですが、△△のケースはどうされてますか？」 |
| 深掘り質問 | 「◯◯の部分が気になりました。△△ではなくこちらを選んだ決め手って何かあったんでしょうか」 |
| データ・補足 | 「これ◯◯でも同様でした。実測だと△△だったので、規模によって変わりそうですね」 |
| 節度ある別視点 | 「なるほどです。逆に◯◯の場合は△△が効いた経験があって、使い分けかもしれません」 |

- **NG**: 「すごい！」「参考になります」だけ／絵文字のみ／自分の宣伝。著者が「返信し返したくなるか」が基準
- 相手から返信が来たら会話を1〜2往復続ける（この往復がアルゴリズム上いちばん重い）

### (3) 引用RT＋ネタ集め — 3〜5分
- 良い技術ポストを1〜2件、**自分の一言（経験・意見）を添えて**引用RT
- タイムラインで気づいたこと・今日ハマったことをメモ → ②のライブネタや次バッチの種にする

## ② ライブネタ投稿：週1（5分）

- その週に**実際あったこと**を1本手動投稿：今日ハマったバグ、いま作っているもの、学習の途中経過
- 完成した話より**途中の話**。「〜で迷っている」「〜という理解で合ってるだろうか」のような
  問いかけで終えると返信が来やすい
- 予約枠と同じ日でも構わない（1日2本になっても問題ない）
- **投稿後の記録**: `post/` に md 化して `status: 予約済み` で push しておく
  → 日曜の自動処理（reconcile）が tweet_id を拾い、指標収集に乗る。整形は `/create-x-post` に頼めばよい

---

## ③ 日曜バッチ（20〜30分）

1. `git pull` → **`analysis/` の最新週次レポートを読む**（日曜11時に自動生成済み）
2. レポートの改善提案を見て、翌週の出し方を決める
3. 在庫レビュー：次に出す下書きを確認（**現在庫は7/25で尽きる** → `/create-x-post` で補充）
4. 出すものの frontmatter を `status: 承認済み`＋`scheduled_at`（9/12/15/18/20/22時をばらす）にして **push** ＝予約完了
5. `reply_targets.md` の手入れ：反応をくれた人を追加、2週間反応がない人を入れ替え

---

## ④ 月1：Premiumアナリティクス確認（10分）

- Xアプリ → プレミアム → アナリティクス で **プロフィール訪問数（アクセス）とフォロワー増減**を確認
  （この2つはAPIで取れないため自動レポートに含まれない）
- 「プロフィール訪問は増えているのにフォローが増えない」→ bio・固定ツイートを見直す合図（`account.md` の確定版と見比べる）
