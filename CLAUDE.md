# X投稿作成ガイドライン

## プロジェクト概要
このプロジェクトはXアカウント(@FujihashiYuya)の投稿文を作成・レビューするためのものです。

## 必須参照ファイル

**投稿の作成・レビュー時は必ず `account.md` を参照すること**

```
参照タイミング:
1. 投稿案の作成前 → ターゲット読者、コンテンツ比率を確認
2. 下書きレビュー時 → NGルール、文体方針をチェック
3. 最終確認時 → コアメッセージとの整合性を確認
```

| ファイル | 内容 | 用途 |
|----------|------|------|
| `account.md` | アカウント運用方針の詳細 | 投稿作成・レビューの基準 |
| `CLAUDE.md` | 作業ガイドライン | ワークフロー・チェックリスト |
| `post/*.md` | 過去の投稿記録 | 参考・重複防止 |

---

## 投稿ファイル出力ルール

### 保存場所
`post/` フォルダ配下にmdファイル形式で保存

### ファイル命名規則
```
YYYYMMDD_連番_テーマ.md
例: 20250115_01_AIエージェント.md
```

### ファイル構成
```markdown
# 投稿文

[投稿本文をここに記載]

---

## メタ情報
- 作成日: YYYY-MM-DD
- カテゴリ: [技術ログ / 学習習慣 / 個人開発 / ツール活用]
- 文字数: XX文字
- ステータス: [下書き / レビュー済み / 投稿済み]
```

### 運用ルール
- 1ファイル = 1投稿（スレッド投稿は1ファイルにまとめる）
- 投稿完了後はステータスを「投稿済み」に更新
- 却下した案も記録として残す（ステータス: 却下）

## アカウント方針サマリー

### ターゲット像
- エンジニアとしての実務経験・個人開発・学習の記録を発信
- 技術的な思考力と実行力を備えた開発者としての信頼構築
- AI・ツール活用、設計・要件定義などの上流思考も含めた技術視点

### 投稿ジャンル
1. 技術ログ・フレームワーク活用
2. 学習習慣・インプットログ
3. 個人開発・副業とのバランス
4. ツール活用と実務効率化

### 文体方針
- 誠実・実直（軽すぎず、堅すぎない）
- 自分の学びや気づきを「共有する」姿勢
- 押しつけない提案型の語り口
- ハッシュタグは原則なし（使う場合も最大2つ）

## 投稿レビューチェックリスト

### 1. アカウント方針との整合性
- [ ] 実体験ベースの内容か
- [ ] 技術的な気づき・学びが含まれているか
- [ ] 誠実・実直なトーンか
- [ ] 押しつけがましくないか
- [ ] コアメッセージ「技術を理解し、考え、形にし続けるエンジニアの記録」に合致するか

### 2. リスクチェック（炎上・情報漏洩防止）
- [ ] 会社名・プロジェクト名が含まれていないか
- [ ] クライアント情報が推測できる内容ではないか
- [ ] 守秘義務に抵触しないか
- [ ] 特定の個人・企業への批判になっていないか
- [ ] 政治的・宗教的な発言が含まれていないか
- [ ] 炎上しやすい表現（断定的すぎる主張、マウント）がないか
- [ ] 個人情報（住所、電話番号、メールアドレス等）が含まれていないか

### 3. フォーマット整形
- [ ] 140文字以内に収まっているか（スレッドの場合は各ツイート）
- [ ] 読みやすい改行が入っているか
- [ ] ハッシュタグは2つ以下か
- [ ] リンクがある場合は適切か
- [ ] 絵文字の使用は控えめか

## 投稿テンプレート

### 技術学習系
```
[学習内容/技術名]について[学んだこと/気づいたこと]。
[具体的なポイント]
[他の人への示唆や共感ポイント]
```

### ツール紹介系
```
[ツール名]を使ってみたら[効果/感想]。
[具体的な使い方やポイント]
[おすすめの対象者]
```

### 個人開発系
```
[開発中のもの/取り組み]について。
[進捗/課題/気づき]
[次のステップや考えていること]
```

## 注意事項
- 副業案件の具体的な内容は投稿しない
- 本業の詳細（会社名、プロジェクト、技術スタック）は伏せる
- 批判より建設的な視点を優先
- 「〜すべき」より「〜と感じた」「〜が良かった」の表現を使う

## 2025年1月 ITエンジニア界隈トレンド

### AIエージェント関連（最注目）
- **2025年は「AIエージェント元年」**: 生成AIが「ツール」から「業務を遂行する主体」へ進化
- **MCP（Model Context Protocol）**: Anthropic社が開発した業界標準プロトコル、様々なサービスとの連携が容易に
- **マルチエージェント連携**: A2A、AGNTCYなどのプロトコル整備が進行中
- **ChatGPT AIエージェント機能**: Operator、Deep Researchなど複雑なワークフローを自動実行

### プログラミング言語トレンド
- **Python**: AI・機械学習需要で圧倒的1位（シェア23%超、前年比+6.67%）
- **Go**: 効率的なシステム構築で企業採用増加
- **TypeScript**: JavaScriptからの移行が業界全体で加速
- **Flutter**: クロスプラットフォーム開発で採用増

### セキュリティ関連
- **ゼロトラスト**: すべての接続を信頼しない設計が必須に
- **Windows10サポート終了**: 2025年10月14日に終了、OS移行が急務
- **サプライチェーン攻撃**: 対策の重要性が増加

### その他の注目トピック
- **2025年の崖問題**: レガシーシステム維持限界、年間12兆円の経済損失予測
- **ノーコード/ローコード**: 導入率51%到達、市民開発が定着
- **量子コンピューティング**: 実用化に向けた進展

### 投稿ネタとして使いやすいテーマ
1. AIエージェントを使った開発効率化の体験
2. MCPを使ったツール連携の試行
3. Python/TypeScript学習の気づき
4. セキュリティ学習の重要性（アカウント既存テーマと一致）
5. レガシーシステムとの向き合い方
